# 05 — Requirements: AI Agent Framework

**Модуль:** `AGENT` · **Статус:** Draft · **Зависит от:** 02, 04, 06 · **Design:** `../design/agent-framework.md`

---

## User Stories

- **US-AGENT-1**: Как владелец канала, я хочу, чтобы канал обслуживала команда
  специализированных агентов (исследователь, планировщик, автор, SEO, ревьюер,
  публикатор, аналитик), чтобы каждый шаг делала подходящая модель.
- **US-AGENT-2**: Как владелец, я хочу, чтобы агент помнил историю канала и реакции
  аудитории, чтобы не повторяться и становиться лучше.
- **US-AGENT-3**: Как автор, я хочу AI Copilot в редакторе, который предлагает
  заголовки, вступления, CTA и SEO прямо во время написания.

## Состав агентов

```
Research Agent → Planning Agent → Writer Agent → SEO Agent
      → Reviewer Agent → Publisher Agent → Analytics Agent
```

Каждый агент использует свою модель (пример: Research → DeepSeek, Writer → GPT-5,
SEO → GPT-5 Mini, Review → Claude) — назначение конфигурируемо (REQ-PIPE-010).

## Functional Requirements (EARS)

### Framework

- **REQ-AGENT-001**: THE SYSTEM SHALL реализовать агентов как композицию шагов
  с единым контрактом: вход (typed context) → инструменты → выход (typed result);
  агенты соединяются в последовательности/DAG пайплайна (REQ-PIPE-040).
- **REQ-AGENT-002**: Каждый агент SHALL иметь доступ только к инструментам из своего
  allowlist (поиск, чтение памяти, генерация, публикация и т.д.); вызов инструмента
  вне allowlist SHALL отклоняться и логироваться.
- **REQ-AGENT-003**: Агент SHALL уметь: искать, рассуждать (многошагово), писать,
  проверять, переписывать, создавать изображения, искать источники, делать SEO,
  создавать теги, публиковать, анализировать — через инструменты, а не монолитный промпт.
- **REQ-AGENT-004**: WHEN агент превышает бюджет шагов или токенов на задачу,
  THE SYSTEM SHALL прервать его с сохранением частичного результата и пометкой
  `budget_exceeded`.

### Специализация агентов

- **REQ-AGENT-010 (Research)**: Research Agent SHALL искать: новости, интересные темы,
  тренды, горячие обсуждения, новые исследования, продукты, законы — по источникам
  канала и внешним поисковым инструментам.
- **REQ-AGENT-011 (Trend Finder)**: THE SYSTEM SHALL ежедневно анализировать
  Google Trends, Telegram, Twitter, Reddit, новости, YouTube, Hacker News, GitHub
  (по мере доступности коннекторов) и предлагать каналу темы с оценкой релевантности
  аудитории.
- **REQ-AGENT-012 (Planner)**: Content Planner SHALL формировать план публикаций
  с рекомендациями типа и времени («сегодня — новость, через два часа — история,
  завтра — большая статья, в пятницу — подборка»), учитывая аналитические данные
  канала (лучшие часы, лучшие типы — REQ-AN-020).
- **REQ-AGENT-013 (Writer)**: Writer Agent SHALL генерировать материал заданного типа
  с учётом Brand Profile, AI Personality, Style Embedding (few-shot из ближайших
  примеров) и фактов с источниками из этапа extraction.
- **REQ-AGENT-014 (Editor/Reviewer)**: AI Editor SHALL проверять: повторы, ошибки,
  галлюцинации (утверждения без источника), тональность, логику, структуру, SEO,
  понятность, CTR-потенциал заголовка — и возвращать структурированный отчёт
  с правками.
- **REQ-AGENT-015 (SEO)**: SEO Agent SHALL формировать: мета-описание, ключевые слова,
  теги/хештеги, alt-тексты изображений, внутренние рекомендации по структуре.
- **REQ-AGENT-016 (Publisher)**: Publisher Agent SHALL публиковать материал согласно
  режиму канала (REQ-WS-014) и адаптировать форматирование под целевую платформу.
- **REQ-AGENT-017 (Analytics)**: Analytics Agent SHALL собирать метрики публикаций
  и формировать выводы для памяти канала (REQ-AGENT-020).

### Channel Memory

- **REQ-AGENT-020**: THE SYSTEM SHALL вести память канала: что публиковалось,
  что нельзя повторять, что понравилось аудитории, что зашло плохо, популярные темы,
  успешные заголовки и структуры.
- **REQ-AGENT-021**: WHEN генерируется новая идея, THE SYSTEM SHALL проверять её по
  памяти (семантическая близость к прошлым публикациям) и отклонять/переформулировать
  повторы моложе настроенного окна (default 90 дней).
- **REQ-AGENT-022**: Память SHALL быть двухслойной: структурированные записи
  (таблицы: публикации, вердикты, темы) + векторный слой для семантического поиска;
  агенты читают память через инструмент `memory.search`, а не сырыми SQL.
- **REQ-AGENT-023**: Пользователь SHALL видеть и редактировать память канала
  (удалить запись, добавить запрет, закрепить предпочтение).

### AI Copilot (Phase 3)

- **REQ-AGENT-030**: WHEN пользователь редактирует материал, THE SYSTEM SHALL
  предлагать по запросу: варианты заголовка, вступления, CTA, идеи изображений,
  SEO-подсказки, FAQ-блок.
- **REQ-AGENT-031**: Предложения Copilot SHALL применяться только явным действием
  пользователя (никакого автоприменения).

## Acceptance Criteria

- [ ] AC-AGENT-1: вызов инструмента вне allowlist агента отклоняется и попадает в лог. *(REQ-AGENT-002)*
- [ ] AC-AGENT-2: идея, семантически совпадающая с публикацией 30-дневной давности, отклоняется как повтор. *(REQ-AGENT-021)*
- [ ] AC-AGENT-3: отчёт Reviewer содержит все категории проверок REQ-AGENT-014 (структурированный JSON). 
- [ ] AC-AGENT-4: превышение бюджета токенов агентом прерывает задачу с `budget_exceeded` и частичным результатом. *(REQ-AGENT-004)*
- [ ] AC-AGENT-5: план публикаций использует историческую аналитику канала (в тесте — фикстурные метрики меняют рекомендованное время). *(REQ-AGENT-012)*
